# lfsr128sum

Высокопроизводительная C++20 библиотека и консольная утилита для вычисления 128-битных контрольных сумм на базе парных LFSR-генераторов (Linear Feedback Shift Register). Проект сочетает классическую математику регистров сдвига с современными SIMD-инструкциями.

## 🚀 Ключевые особенности

*   **Экстремальная производительность**: Скорость обработки данных достигает **~1.6 ГБ/с** на одно ядро (архитектура x86-64 с поддержкой AVX2/SSE4.2).
*   **SIMD-оптимизация**
*   **Математическая обратимость**
*   **Гарантированное качество**: 100% Coverage (покрытие) для 32-битных и 64-битных хэшей на блоке 256 байт, в котором заполняются первые два байта.
*   **Интеллектуальная соль (Salt)**: Учет реального размера файла в "соли" предотвращает коллизии файлов, различающихся только количеством нулевых байтов в конце.
*   **Современный стек**: C++20 (`std::span`, concepts), CMake и кроссплатформенная поддержка (Windows/MSVC, Linux/GCC).

## 🛡 Алгебраическая архитектура (Multi-Field Mixing)

В отличие от стандартных контрольных сумм (например, CRC32), работающих в одном поле $GF(2^n)$, алгоритм `lfsr128sum` построен на «столкновении» трех различных алгебраических структур:

1.  **Поле $GF(251^4)$**: Арифметика полиномов над простым модулем 251.
2.  **Поле $GF(241^4)$**: Арифметика полиномов над простым модулем 241.
3.  **Операция XOR**: Смешивание состояний, выходящее за рамки обоих полей.

## 🛡️ Анализ криптографической стойкости

Архитектура системы основана на четырех независимых генераторах состояний, объединенных операцией $XOR$. Выбор параметров минимизирует математический резонанс и максимизирует итоговый период хеширования.

### 1. Параметры генераторов
Система использует два простых основания $p \approx 2^8$ и взаимно простые степени $n \in \{3, 4\}$:
*   **Группа A:** $p_1 = 251$, периоды $T_{A1} = 251^4 - 1$, $T_{A2} = 251^3 - 1$
*   **Группа B:** $p_2 = 241$, периоды $T_{B1} = 241^4 - 1$, $T_{B2} = 241^3 - 1$

### 2. Математическое обоснование периода
Общий период системы $T_{total}$ вычисляется как наименьшее общее кратное (lcm) всех четырех периодов:

$$T = \text{lcm}(\text{lcm}(251^4-1, 251^3-1), \text{lcm}(241^4-1, 241^3-1))$$
$$T \approx 6.86 \times 10^{25} \gtrsim 2^{85.8}$$

**Почему это эффективно:**
*   **Взаимная простота степеней (3 и 4):** Поскольку $\text{gcd}(3, 4) = 1$, полиномы $x^3-1$ и $x^4-1$ имеют минимальное пересечение — только через общий множитель $(x-1)$. Если бы использовались степени $2$ и $4$, период сократился бы в $x^2-1$ раз (для 251 это потеря в $\approx 63\,000$ раз).
*   **Простые основания (241 и 251):** Использование разных простых чисел гарантирует, что специфические множители периодов (такие как $31501$ и $29041$) не пересекаются, что обеспечивает мультипликативный рост итогового цикла.

### 3. Стойкость к коллизиям
С суммарным периодом $\approx 2^{86}$, пространство состояний хеш-функции превосходит стойкость классических 64-битных алгоритмов. Динамическая соль, зависящая от длины сообщения $L$, гарантирует уникальность траектории генераторов даже при идентичном начале данных.

```

## 🛠 Сборка проекта

Проект не имеет внешних зависимостей. Для сборки требуется компилятор с поддержкой C++20.

```bash

## 🖥 Использование

### Консольная утилита
Программа работает как классический хэш-сумматор. Поддерживаются режимы вычисления хэша, тестирования и замера скорости.

*   **Вычисление хэша файла**:
    ```bash
    ./lfsr128sum <путь_к_файлу>
    ```
*   **Запуск тестов на покрытие и обратимость**:
    ```bash
    ./lfsr128sum --test
    ```
*   **Запуск встроенного бенчмарка производительности**:
    ```bash
    ./lfsr128sum --bench
    ```

### Использование как C++ библиотеки
Библиотека поддерживает **поточное хэширование** (Streaming Mode), что позволяет обрабатывать файлы любого объема, сохраняя зависимость каждого последующего блока от предыдущего состояния генератора.

```cpp
#include "lfsr_hash.h"

// 1. Инициализация генератора
lfsr_hash::gens generator;
generator.reset(); // Сброс в начальное состояние {1, 0...}

// 2. Потоковая обработка данных (например, в цикле чтения файла)
// Состояние генератора сохраняется между вызовами hash128
for (auto& chunk : file_chunks) {
    // Внутри hash128 вызывается g.process_input() и подмешиваются соли,
    // но НЕ вызывается reset(), что обеспечивает цепочечную зависимость.
    lfsr_hash::u128 block_hash = lfsr_hash::hash128(generator, chunk);
    
    // Итоговый хэш можно формировать через XOR промежуточных результатов
    final_hash.first  ^= block_hash.first;
    final_hash.second ^= block_hash.second;
}

// 3. Математическая обратимость (для библиотечных нужд)
// Откат состояния ровно на 4 такта назад.
generator.g_251x4.back_simd_block(last_input_block);
