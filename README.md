# lfsr128sum (v2.1)

Высокопроизводительная C++20 библиотека и консольная утилита для вычисления 128-битных контрольных сумм на базе парных LFSR-генераторов (Linear Feedback Shift Register). Проект сочетает классическую математику регистров сдвига с современными SIMD-инструкциями.

## 🚀 Ключевые особенности

*   **Экстремальная производительность**: Скорость обработки данных достигает **~1.6 ГБ/с** на одно ядро (архитектура x86-64 с поддержкой AVX2/SSE4.2).
*   **SIMD-оптимизация**: Векторная обработка 8 ячеек состояния одновременно с использованием `_mm_alignr_epi8`, `_mm_mullo_epi16` и быстрой редукции Барретта.
*   **Математическая обратимость**: Уникальная поддержка функций `back_simd_block`, позволяющих "отматывать" состояние генератора назад на 128 бит за один шаг.
*   **Гарантированное качество**: 100% Coverage (покрытие) на 64-битных тестах. Высокая диффузия входных данных исключает коллизии на коротких дистанциях.
*   **Интеллектуальная соль (Salt)**: Учет реального размера файла в соли предотвращает коллизии для файлов, различающихся только количеством нулевых байтов в конце.
*   **Современный стек**: C++20 (`std::span`, `std::filesystem`, concepts), CMake и кроссплатформенная поддержка (Windows/MSVC, Linux/GCC).

## 🛡 Алгебраическая архитектура (Multi-Field Mixing)

В отличие от стандартных контрольных сумм (например, CRC32), работающих в одном поле $GF(2^n)$, алгоритм `lfsr128sum` построен на «столкновении» трех различных алгебраических структур:

1.  **Поле $GF(251^4)$**: Арифметика полиномов над простым модулем 251.
2.  **Поле $GF(241^4)$**: Арифметика полиномов над простым модулем 241.
3.  **Операция XOR**: Нелинейное смешивание состояний, выходящее за рамки обоих полей.

### Анализ периодов
Периоды генераторов составляют $T_1 = 251^4 - 1$ и $T_2 = 241^4 - 1$. 
Математический анализ показывает, что **$\gcd(T_1, T_2) = 240$**. 

Это крайне низкое значение относительно общего пространства состояний ($1.5 \times 10^{19}$) гарантирует:
*   **Гигантский суммарный цикл**: Общий период комбинации равен $LCM(T_1, T_2) \approx 1.5 \times 10^{17}$ тактов.
*   **Стойкость к алгебраическим атакам**: Несовместимость модулей (251 vs 241 vs XOR) делает невозможным сведение системы уравнений для восстановления состояния к единому полю.


## 🛠 Сборка проекта

Проект не имеет внешних зависимостей. Для сборки требуется компилятор с поддержкой C++20.

```bash
# Клонирование
git clone https://github.com
cd lfsr128sum

# Сборка в режиме Release (важно для SIMD оптимизаций)
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
```

## 🖥 Использование

### Консольная утилита
Программа работает как классический хэш-сумматор. Поддерживаются режимы вычисления хэша, тестирования и замера скорости.

*   **Вычисление хэша файла**:
    ```bash
    ./lfsr128sum <путь_к_файлу>
    ```
*   **Запуск тестов на покрытие и обратимость**:
    ```bash
    ./lfsr128sum --test
    ```
*   **Запуск встроенного бенчмарка скорости (MiB/s)**:
    ```bash
    ./lfsr128sum --bench
    ```

### Использование как C++ библиотеки
Библиотека поддерживает **поточное хэширование** (Streaming Mode), что позволяет обрабатывать файлы любого объема, сохраняя зависимость каждого последующего блока от предыдущего состояния генератора.

```cpp
#include "lfsr_hash.h"

// 1. Инициализация генератора
lfsr_hash::gens generator;
generator.reset(); // Сброс в начальное состояние {1, 0...}

// 2. Потоковая обработка данных (например, в цикле чтения файла)
// Состояние генератора сохраняется между вызовами hash128
for (auto& chunk : file_chunks) {
    // Внутри hash128 вызывается g.process_input() и подмешиваются соли,
    // но НЕ вызывается reset(), что обеспечивает цепочечную зависимость.
    lfsr_hash::u128 block_hash = lfsr_hash::hash128(generator, chunk);
    
    // Итоговый хэш можно формировать через XOR промежуточных результатов
    final_hash.first  ^= block_hash.first;
    final_hash.second ^= block_hash.second;
}

// 3. Математическая обратимость (для библиотечных нужд)
// Откат состояния ровно на 1 такт (16 байт) назад
generator.g_251x4.back_simd_block(last_input_block);
